\section{Судалгааны ерөнхий загвар}

Энэхүү судалгааны ажлын хүрээнд эхний ээлжид симуляцийн орчныг хөгжүүлсэн болно. Үүний дараа тэмцээний даалгаврыг гүйцэтгэхэд шаардлагатай объектуудыг таних зорилгоор, эдгээр объектууд болон камерын харагдах хүрээнд өртөх боломжтой төрөл бүрийн нөхцөлүүдийг (жишээлбэл, гэрэлтүүлэг, сүүдэр, камерын хөдөлгөөн зэрэг) хамарсан датасет үүсгэх программыг хөгжүүлсэн.

Үүсгэсэн датасет дээр үндэслэн объект таних алгоритмыг сургаж, улмаар сургасан загварыг симуляцийн орчинд туршин баталгаажуулсан болно.

\section{Симуляцийн орчин}

\subsection{Модель үүсгэх}

Симуляцийн орчныг бүрдүүлэх зорилгоор тэмцээний талбай болон ашиглагдах элементүүдийг гурван хэмжээст загварчлах программ ашиглан боловсруулсан.

Тэмцээний талбайн моделиудыг тэмцээний дүрэмд хавсаргасан техникийн зурагт тулгуурлан боловсруулсан бөгөөд, геометрийн нарийвчлал шаардсан загварчлалд FreeCAD программ илүү тохиромжтой гэж үзэн ашигласан. Харин материаллаг шинж чанар, дүрслэлийн чанарыг сайжруулах зорилгоор Blender программыг ашигласан.

\subsection{Unity орчин үүсгэх}

Unity нь өндөр чанартай, бодитой симуляцийн орчин бүрдүүлэх боломжийг олгодог бөгөөд, нэмэлтээр хэрэглэгчийн тодорхойлсон программын логикийг хэрэгжүүлэх боломжтой. Иймд энэхүү судалгаанд уг орчныг зөвхөн симуляцийн зориулалтаар бус, датасет боловсруулахад мөн ашигласан болно.

\subsubsection{Загвар импортлох ба орчин бүрдүүлэх}

Blender болон FreeCAD программуудад боловсруулсан 3D загваруудыг \textit{FBX} өргөтгөлтэйгөөр экспортлон Unity-д импортолсон. FBX формат нь геометр, материал, текстурийн мэдээллийг хамтад нь хадгалдаг тул загваруудын харагдах байдал алдагдалгүй шилжих боломжийг олгодог.

Импортолсон загваруудыг тэмцээний талбайн зурагт нийцүүлэн байршуулсны дараа тус бүрд нь \textit{Collider} бүрэлдэхүүн нэмсэн. Collider нь объектуудын хоорондох физикийн харилцан үйлчлэлийг тодорхойлдог бөгөөд, талбайн хана, шал болон бусад саад тотгорт \textit{Box Collider} ашигласан.

\subsubsection{Тоглогч болон камерын тохиргоо}

R1 роботын хөдөлгөөнийг дүрслэх зорилгоор хэрэглэгчийн оролтоор удирдагдах тоглогчийн объект үүсгэсэн. Тоглогчийн объектын дотор \textit{Camera} бүрэлдэхүүн байршуулснаар роботын харах өнцгөөс симуляцийг ажиглах боломжтой болсон.

Үүнээс гадна датасет боловсруулах зориулалттай нэмэлт камерыг тусад нь байршуулсан. Энэхүү камер нь тоглогчоос үл хамааран тогтмол байрлалд байрших бөгөөд, объект таних алгоритмыг сургахад шаардлагатай зургуудыг авахад ашигладаг.

\section{Датасет боловсруулах}

Объект таних алгоритмыг үр дүнтэй сургахад их хэмжээний, олон янзын нөхцөлийг хамарсан датасет шаардлагатай. Бэлэн датасетууд судалгааны зорилгод бүрэн нийцээгүй тул симуляцийн орчинд суурилан датасетийг үүсгэсэн.

Хэдийгээр ийм аргаар үүсгэсэн датасет нь бодит орчны бүх нөхцөлийг бүрэн тусгаж чадахгүй боловч, симуляцийн орчинг бодит нөхцөлд ойртуулах замаар ерөнхий тохиолдолд объект таних чадварыг хангалттай түвшинд сургах боломжтой.

Датасет үүсгэх системийн ерөнхий архитектурыг Зураг \ref{fig:dataset-generation-architecture}-д үзүүлэв. Үр дүнтэй датасет бүрдүүлэхийн тулд камер болон объектуудын байрлалыг санамсаргүй байдлаар өөрчилж, олон өнцгөөс зураг авах шаардлагатай. Мөн орчны гэрэлтүүлгийг өөрчлөх замаар алгоритмыг зөвхөн нэг төрлийн нөхцөлд хэт дасахаас сэргийлсэн.

Зураг авсны дараа тухайн зураг бүрт харгалзах тайлбар \textit{(label)} файлыг үүсгэнэ. Unity орчин нь объектуудын дэлхийн координатыг мэддэг тул эдгээрийг камерын координатын системд хөрвүүлэх замаар дүрс дээрх байршлыг тооцоолох боломжийг олгодог.

\subsection{Датасет тайлбар үүсгэх}

Зураг~\ref{fig:label-generation-algorithm}-д үзүүлсэн псевдокод нь объектийн оройн координатуудыг камерын координатын системд хөрвүүлж, дэлгэцэн дээрх проекцийн хамгийн бага болон хамгийн их координатыг тодорхойлдог. Тооцоолсон координатуудаар тогтоогдох хязгаарлах тэгш өнцөгт \textit{(bounding box)} нь тухайн объектийн тайлбар болно. Харин объект камерын харагдах хүрээнд ороогүй тохиолдолд тайлбар үүсгэхгүй.

\begin{figure}[H]
\begin{minted}{python}
def generate_label(object):
    if object is not visible: # Хэрэв объект харагдахгүй бол тайлбар үүсэхгүй
        return None
    vertices = object.vertices # Объектийн оройн координатууд
    minimum = None # Хамгийн бага цэг
    maximum = None # Хамгийн их цэг
    for vertex in vertices:
        # Камерийн координат руу шилжүүлнэ
        vertex2 = camera.world_to_screen_point(vertex)
        # Хэрэв хамгийн бага цэг нь тодорхойлогдоогүй эсвэл vertex2 нь хамгийн бага цэг бол хамгийн бага цэгийг шинэчлэнэ
        if minimum is None or vertex2 < minimum:
            minimum = vertex2
        # Хэрэв хамгийн их цэг нь тодорхойлогдоогүй эсвэл vertex2 нь хамгийн их цэг бол хамгийн их цэгийг шинэчлэнэ
        if maximum is None or vertex2 > maximum:
            maximum = vertex2
    # Эцэст нь хамгийн бага болон хамгийн их цэгийн хүрээнд үүсэх дөрвөлжин нь объектийн зааг болно.
    return rectangle(minimum, maximum)
\end{minted}
\caption{Тайлбар үүсгэх алгоритмын псевдокод}
\label{fig:label-generation-algorithm}
\end{figure}

\subsection{Камер байршуулах}

Камер нь таних объектуудыг ерөнхийд нь харсан байх бөгөөд, мөн байж болох төрөл бүрийн өнцгөөс харсан байвал зохино. Үүний тулд камерыг санамсаргүй байдлаар байршуулах алгоритм хэрэгжүүлсэн.

Датасет боловсруулах үед тоглогчийн объект нь таних объектуудын ерөнхий төвд байрлана. Камер нь тоглогчийг голлон чиглэсэн байх бөгөөд, байршлыг тодорхойлохдоо дараах дарааллыг баримтална:

\begin{enumerate}
    \item \textbf{Зай тохируулах} --- Камерыг тоглогчоос өгөгдсөн хамгийн бага болон хамгийн их хүрээний хооронд санамсаргүй зайд байрлуулна.
    \item \textbf{Фокуслах} --- Камерыг тоглогч объект луу чиглүүлнэ.
    \item \textbf{Эргүүлэх} --- Камерыг гурван тэнхлэгийн дагуу санамсаргүй өнцгөөр эргүүлнэ:
    \begin{itemize}
        \item $Y$ тэнхлэг: $0^\circ$-с $360^\circ$ (бүх чиглэлийг хамрах)
        \item $X$ тэнхлэг: $-30^\circ$-с $+30^\circ$ (дээрээс болон доороос харах өнцөг)
        \item $Z$ тэнхлэг: $-20^\circ$-с $+20^\circ$ (налуу өнцөг)
    \end{itemize}
    \item \textbf{Байрлалын шилжилт \textit{(offset)} нэмэх} --- Камерын эцсийн байрлалд баруун, зүүн, дээш, доош чиглэлд санамсаргүй шилжилт нэмж, байршлыг нарийвчлан тохируулна.
\end{enumerate}

Энэхүү аргаар камерыг олон янзын байрлал, өнцгөөс байршуулснаар объектуудыг өргөн хүрээний нөхцөлд хамарсан датасет бүрдүүлэх боломжтой болно.

\section{Объект таних алгоритм}

Объект таних алгоритмыг хэрэгжүүлэхэд YOLOv12 загварыг сонгон ашигласан. Сургалтад симуляцийн орчинд боловсруулсан датасетийг ашигласан бөгөөд, сургасан загварыг симуляцийн орчинтой холбон туршсан.

\subsection{Сургалт}

Сургалтыг YOLOv12 загварын урьдчилан сургасан жин \textit{(pre-trained weights)} дээр суурилан шилжүүлэн сургах \textit{(transfer learning)} аргаар явуулсан. Энэ аргын давуу тал нь ерөнхий дүрс таних чадварыг хадгалан, тэмцээний тусгай объектуудад тохируулан нарийвчлах боломж олгодогт оршино. Сургалтад симуляцийн орчинд үүсгэсэн датасетийг ашигласан бөгөөд, датасет нь олон янзын гэрэлтүүлэг, байрлал, өнцгийн нөхцөлийг хамарсан байна.

\subsection{HTTP сервер}

Сургасан загварыг симуляцийн орчинд туршихын тулд Python хэл дээр HTTP сервер үүсгэсэн. Серверийн үндсэн үүрэг нь симуляцийн орчин болон объект таних алгоритмын хооронд зуучлах явдал юм. Үйл ажиллагааны дараалал дараах байдалтай:

\begin{enumerate}
    \item Симуляцийн орчин камерын зургийг HTTP хүсэлтээр серверт илгээнэ.
    \item Сервер хүлээн авсан зургийг YOLOv12 загварт дамжуулан боловсруулна.
    \item Загвараас гарсан үр дүнг серверээс симуляцийн орчинд буцаан илгээнэ.
    \item Симуляцийн орчин хүлээн авсан үр дүнд үндэслэн таньсан объект бүрийн нэр болон хязгаарлах тэгш өнцөгтийг дэлгэцэн дээр дүрсэлнэ.
\end{enumerate}

\subsection{Дата дамжуулалт}

Сервер нь таньсан объект бүрийн байршил, хэмжээ болон ангиллын дугаарыг дараалласан түүхий дата \textit{(raw binary data)} хэлбэрээр буцаана. Анхны хэрэгжүүлэлтэд JSON форматыг ашиглаж туршсан боловч, хоёр форматыг харьцуулахад дараах ялгаа ажиглагдсан:

\begin{itemize}
    \item JSON дата: дунджаар $\sim$2000 байт
    \item Түүхий дата: дунджаар $\sim$200 байт
\end{itemize}

Түүхий дата нь JSON форматтай харьцуулахад дунджаар 10 дахин бага хэмжээтэй байсан. Нэмж дурдвал, түүхий датаг боловсруулах нь JSON задлах үйлдлээс хэрэгжүүлэлтийн хувьд энгийн байсан тул түүхий дата форматыг эцсийн шийдэл болгон ашиглахаар болсон.

\section{Шийдвэр гаргах алгоритм}

\subsection{Төлөвийн диаграм}

Шийдвэр гаргах алгоритмыг хязгаарлагдмал төлөвт автомат \textit{(Finite State Machine, FSM)} аргаар хэрэгжүүлсэн. FSM нь урьдчилан тодорхойлогдсон төлөвүүд болон тэдгээрийн хоорондох шилжилтийн нөхцөлд үндэслэн ажилладаг тул роботын үйл ажиллагааны дарааллыг тодорхой, удирдахад хялбар байдлаар илэрхийлэх боломжийг олгодог. Төлөвийн диаграмыг Зураг \ref{fig:state-diagram}-д харуулсан болно. Диаграм нь ерөнхий 4 үндсэн төлөвт хуваагдана:

\begin{enumerate}
    \item \textbf{Тохиргоо} --- Баг болон судрын байршлуудыг тохируулах төлөв. Энэ төлөвт багийн өнгө болон судруудын байршлыг гараар тохируулна.
    \item \textbf{Зэвсэг угсрах} --- Жадны үзүүрийг барьж, R1 робот угсрахыг хүлээх төлөв. Угсралт бүрэн дуусмагц R1 робот зэвсгийг салган авах бөгөөд, энэ үйлдлийг мэдрэх хүртэл хүлээнэ.
    \item \textbf{Судар авах} --- Мэйхуа ойд орж судар авч гарах төлөв. Орох болон гарахдаа R2 роботын зөвшөөрөгдсөн хөдөлгөөний хүрээнээс хэтрэхгүй байхыг шаардана.
    \item \textbf{Судар байршуулах} --- Аренад орж хоосон нүд олоод судар байршуулах төлөв. Үйлдэл дууссаны дараа давталт үргэлжлэх бөгөөд, жадны үзүүр илрүүлсэн тохиолдолд зэвсэг угсрах төлөвт, үгүй бол судар авах төлөвт шилжинэ.
\end{enumerate}
